# TasksBoard

個人利用を想定したタスク管理アプリです。
カテゴリや優先度でタスクを整理し、ダッシュボードで進捗を確認することができます。

## URL

https://tasksboard-app.vercel.app/

## 機能

- ダッシュボード（完了率・カテゴリ別タスク数・優先度別未完了数・未完了タスク一覧）
- タスクの作成・編集・削除
- タスク一覧（カテゴリ・優先度・ステータスによる絞り込み）
- タスク一覧のソート（カテゴリ・優先度・ステータス）
- タスク一覧のページネーション
- カテゴリ名の編集（設定画面）

## 採用技術と選定理由

| 技術 | 選定理由 |
|------|---------|
| Next.js (App Router) | Reactベースのフレームワークの中でも標準的な構成であり、個人学習での経験もあったため採用しました。|
| Tailwind CSS | 実務や個人学習で経験があったため採用しました。HTMLのクラス名だけでスタイルを書けるため、CSSファイルを別途管理する手間が省けますし、`md:` などのプレフィックスでレスポンシブ対応もシンプルに書けるところを評価しています。 |
| localStorage | 今回の要件ではサーバーを用意せずにデータを保存できるlocalStorageが最もシンプルだと判断して採用しました。データ処理をカスタムフックに集約しており、将来データベースに移行する場合もUI側のコードを変えずに対応できる設計にしています。 |
| Vercel | こちらも個人利用の経験があったため採用しました。Next.jsの開発元が提供するホスティングサービスで、GitHubへのプッシュだけで自動デプロイされCI/CDの手間が省けて楽な点を評価しています。 |

## 工夫した点

**型定義・定数の一元管理**

`src/types/index.ts` に型定義と表示用の定数（優先度ラベル・カテゴリカラーなど）をまとめています。優先度やカテゴリの色・表示名を直接文字列で書かず、定数を参照する形にすることで、変更が必要な場合も1ファイルの修正で全体に反映されます。

**フィルタ・ソート・ページネーションの組み合わせ**

タスク一覧では、テキスト検索・カテゴリ・優先度・ステータスの絞り込みに加え、ソートとページネーションを組み合わせて動作するよう実装しました。絞り込み条件やソートが変わった際はページが自動的に1に戻るようにしており、意図しない空ページが表示されないよう考慮しています。

**削除操作の確認ダイアログ**

タスクの削除は取り消しができない操作のため、削除ボタンを押しただけでは削除されず、確認ダイアログを挟む設計にしました。誤操作によるデータ消失を防ぐための配慮です。

**レスポンシブ対応**

PC・スマートフォン両方での操作を考慮し、PCではサイドバー、スマートフォンではボトムナビゲーションを表示する構成にしました。なお、デプロイ環境のiOS Safariではブラウザのツールバー表示に一部課題が残っており、ローカル環境で動作確認いただけると幸いです。

## セットアップ手順

```bash
npm install
npm run dev
```

ブラウザで http://localhost:3000 を開いてください。

ログインパスワードは `task1234` です。

## 既知の問題

スマートフォンのブラウザ（iOS Safari）でデプロイ環境にアクセスした際、ブラウザのツールバーが表示されボトムナビがツールバーに隠れてしまう場合があります。ローカル環境では正常に動作することを確認しています。
