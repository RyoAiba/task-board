# TasksBoard

個人利用を想定したタスク管理アプリです。
カテゴリや優先度でタスクを整理し、ダッシュボードで進捗を確認することができます。

## URL

https://task-app-ts.vercel.app/

ログインパスワード：`task1234`

## 機能

- ダッシュボード（完了率・カテゴリ別タスク数・優先度別未完了数・未完了タスク一覧）
- タスクの作成・編集・削除
- タスク一覧（カテゴリ・優先度・ステータスによる絞り込み）
- タスク一覧のソート（カテゴリ・優先度・ステータス）
- タスク一覧のページネーション
- カテゴリ名の編集（設定画面）

## 採用技術と選定理由

| 技術 | 選定理由 |
|------|---------|
| Next.js (App Router) | Reactベースのフレームワークとして標準的な構成であり、個人学習での経験もあったため採用しました。フォルダ構造がそのままURLになるApp Routerは直感的で保守しやすい点を評価しています。 |
| TypeScript | 型定義によってデータ構造が明確になり、変更やリファクタリングを安全に行えるため採用しました。型の一元管理により、修正が必要な箇所をすぐに特定できます。 |
| Tailwind CSS | 実務や個人学習での経験があったため採用しました。クラス名だけでスタイルを書けるためCSSファイルを別途管理する手間が省けますし、レスポンシブ対応もシンプルに書けるところを評価しています。 |
| localStorage | サーバーを用意せずにデータを保存できるlocalStorageが今回の要件に最もシンプルだと判断して採用しました。データ処理をカスタムフックに集約しており、将来データベースに移行する場合もUI側のコードを変えずに対応できる設計にしています。 |
| Vercel | 個人利用での経験があったため採用しました。GitHubへのプッシュだけで自動デプロイされるためCI/CDの手間が省けて楽な点を評価しています。 |

## 工夫した点

**Context APIによるカテゴリ状態管理**

カテゴリデータはContext APIで管理し、どの画面からでも同一のデータにアクセスできる設計にしました。将来的にタスクデータも同じ構成に移行しやすいよう、カスタムフックとContextをセットで管理しています。

**型定義・定数の一元管理**

`types/index.ts` に型定義と表示用の定数（優先度ラベル・カテゴリカラーなど）をまとめています。優先度やカテゴリの色・表示名を直接文字列で書かず定数を参照する形にすることで、変更が必要な場合も1ファイルの修正で全体に反映されます。

**useMemoによるフィルタ・ソート処理の最適化**

タスク一覧のテキスト検索・カテゴリ・優先度・ステータスの絞り込みとソート処理を`useMemo`でまとめています。関連する値に変化がない限り再計算が走らないため、不要な処理を抑えています。絞り込み条件が変わった際はページが自動的に1に戻るよう制御しており、意図しない空ページが表示されないよう考慮しています。

**フォームのバリデーション**

タスクの作成・編集画面では、タスク名の空文字チェックとカテゴリ未選択のチェックをフォーム送信時に実行し、エラーメッセージをフィールド直下に表示します。タスク名は最大50文字とし、入力欄に文字数カウンターを表示しています。

**削除操作の確認ダイアログ**

タスクの削除は取り消しができない操作のため、削除ボタンを押しただけでは削除されず、確認ダイアログを挟む設計にしました。誤操作によるデータ消失を防ぐための配慮です。

**レスポンシブ対応**

PC・スマートフォン両方での操作を考慮し、PCではサイドバー、スマートフォンではボトムナビゲーションを表示する構成にしました。

## セットアップ手順

```bash
npm install
npm run dev
```

ブラウザで http://localhost:3000 を開いてください。

## 既知の問題

スマートフォンのブラウザ（iOS Safari）でデプロイ環境にアクセスした際、ブラウザのツールバーによってボトムナビが隠れてしまう場合があります。ローカル環境では正常に動作することを確認しています。
